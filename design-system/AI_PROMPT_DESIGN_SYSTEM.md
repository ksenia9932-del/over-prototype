В этой папке находится дизайн-система проекта (цвета, отступы, шрифты, скругления) в виде готовых CSS-файлов, а также исходники (JSON-файлы и шрифты) и инструкции для AI-агентов.

## Структура
- `variables.css` — **главный файл со всеми сгенерированными токенами (CSS-переменные). Включает в себя импорт шрифтов.**
- `fonts.css` — файл с правилами `@font-face` для подключения корпоративного шрифта "TT Norms Pro Tochka".
- `TT Norms Pro Tochka/` — файлы корпоративного шрифта (форматы `.woff` и `.woff2` для web).
- JSON-файлы (`color tokens.json`, `spacings.json`, `typography.json`, `roundings.json`) — сырые данные (source of truth). Используются для генерации CSS или для интеграции с другими платформами.

## Как использовать это в коде
Если вы реализуете React/Vite-приложение или верстаете макеты, вам необходимо:
1. Подключить файл `variables.css` глобально в проекте (например, импортировать в `src/main.tsx` или в главный стилевой файл проекта). Это автоматически подключит и шрифты, и все переменные.
2. Использовать CSS-переменные для стилизации. Для цветов использовать формат `var(--primitive-brand)`, для отступов `var(--spacing-4x)`, для шрифтов `var(--ttn-400-xl-01-size)`.
3. Строго следовать токенам при написании стилей компонентов, не выдумывать значения из головы (hardcode). Если в макете используется отступ в 16px, пишите `var(--spacing-4x)`.
4. Для типографики используйте связку переменных `font-size` и `line-height` согласно жирности и названию стиля (пример: `var(--ttn-500-m-01-size)` и `var(--ttn-500-m-01-lh)`).

> **Важно для AI-агента:** При генерации кода по макетам из Figma через MCP всегда сверяйтесь с файлом `variables.css`. Макет передает вам структуру и абстрактные значения (например, "Primary" или отступ "16px"), вы **обязаны** использовать соответствующие им готовые CSS-переменные, объявленные в `:root`, а не писать "сырые" значения вроде `#191919` или `16px`.
